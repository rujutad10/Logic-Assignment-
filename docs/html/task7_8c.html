Detailed Description

Provides functionality to check CNF clause validity.

This module checks if clauses within a CNF formula are tautologies, i.e., they contain both a literal and its negation.

Algorithm: Recursive descent parser

Collect positive and negative literals per clause Check for complementary pairs (x and ~x)

Time Complexity: O(n)

  • n = total nodes in CNF tree
  • m = number of clauses, c = avg clause size
  • Overall: O(m Ã— c) = O(n)
  • Single traversal with stack-based literal collection

Space Complexity: O(n)

  • c = maximum clause size
  • Literal arrays: O(c)
  • Stack for traversal: O(c)
  • No exponential blow-up like Task 6

int isClauseValid ( Node * clause)

Checks whether a single CNF clause is valid.

Checks if a single clause is valid (a tautology). A clause is valid if it contains both an atom and its negation (e.g., x1 and ~x1).

A clause is valid if it contains both a variable and its negation (e.g., A and ~A). Such clauses always evaluate to True.
